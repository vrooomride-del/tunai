# 폰용 Pro 모드 가능성 설계 검토

작성일: 2026-06-20  
상태: 의견 문서 (구현 아님)

---

## 1. 기술적 가능성 평가

### 1-1. DSP 명령 레이어 — 재사용 가능

모바일과 Pro는 `DspAdapter` 추상 인터페이스를 공유한다. 오늘 크로스오버 작업에서
`writeCrossover → adapterFactory → BLE sendRawFrame`이 모바일에서 그대로 동작했으므로,
핵심 DSP 명령 세트(`writeBiquad`, `writeCrossover`, `writeDelay`, `writeGain`)는
양쪽에서 재사용 가능하다.

**결론: DSP 명령 레이어는 추가 작업 없이 모바일에서도 쓸 수 있다.**

### 1-2. PEQ 상태 관리 — Pro 전용, 이식 부담 있음

Pro의 `PeqController`(`peq_controller.dart`) + `BiquadFilter` 모델은 Pro의
`dsp_engine.dart`에 강하게 결합돼 있다. 모바일에는 이 레이어가 전혀 없다.

- `BiquadFilter`, `FilterType`, `BiquadCoefficients` — Pro-only
- `peqProvider` (StateNotifierProvider) — Pro-only
- `DspEngine.calculateBiquad()` — Pro-only

이식하려면 `dsp_engine.dart`를 shared 패키지로 분리하거나 모바일에 복사해야 한다.
공유 패키지 분리는 모노레포 구조 변경이 필요(M 사이즈).

### 1-3. 연결 경로 — 다름, 但 BLE 측은 모바일이 더 완성됨

| | 모바일 | Pro |
|---|---|---|
| 주 연결 | BLE (ICP5, UUID fff0/fff2) | USB-UART (`flutter_libserialport`) |
| 보조 | — | BLE (일부 지원) |

Pro의 USB-UART 경로는 모바일(Android/iOS)에서 쓸 수 없다. 폰에서 DSP 보드에
접근하려면 BLE만 가능 — 이미 모바일에 구현돼 있으므로 연결 경로 자체는 문제없다.

### 1-4. 측정 정밀도 — 솔직한 한계

폰 내장 마이크는 `MicCalibrationDb`로 기종별 보정 중이지만, 실질적으로:

- **주파수 특성**: 4kHz 이상 급격한 롤오프, 10kHz 이상 신뢰도 낮음
- **다이나믹 레인지**: 측정 마이크(UMIK-1 등) 대비 20~30dB 좁음
- **공간 노이즈 민감도**: 측정 마이크는 삼각대+무반향 설정이 가능하지만 폰은 불가

"Pro급 측정"을 마케팅 문구로 쓰면 과장이다. 폰 마이크로 할 수 있는 건
**"룸 공진 보정(100~2000Hz 대역 피크 제거)"** 수준이고, 이것도 잘 하는 것이
현실적 목표다. 트위터 주파수 대역(4kHz↑) 측정은 폰 마이크로 의미 있는 데이터를
뽑기 어렵다.

## 2. 제품 방향 3안

### 안 A — "고급 모드 토글" (기존 앱 내 숨김 메뉴)

기존 모바일 앱 설정 화면에 **"Pro 기능 활성화"** 토글을 추가. 활성화 시
홈 화면에 수동 PEQ 패널이 노출된다. 라이선스 검증(서버 API)으로 유료 잠금 가능.

**장점**
- 앱 하나로 라이트/프로 구분 → 마케팅 단순
- 기존 BLE 파이프 그대로 재사용
- 유료 인앱 업그레이드로 매출 경로 추가 가능

**단점**
- 폰 화면에서 20밴드 PEQ 조작은 UX 한계(아래 2-2 참고)
- Pro 앱 라이선스(₩99,000) 가치가 희석될 수 있음
- 코드 복잡도 증가: 모바일에 dsp_engine 이식 필요 (M 사이즈)

**T-shirt 사이즈: M** (dsp_engine 이식 + 토글 UI + 라이선스 게이트)

---

### 안 B — "Pro Lite 탭" (별도 탭, 제한된 기능만)

홈 화면에 탭을 하나 추가해 **수동 PEQ 5밴드** + **크로스오버 수동 조정**만 노출.
20밴드 전체가 아닌 5밴드 파라메트릭으로 한정 — 폰 화면에서 쓸 수 있는 현실적 범위.
FRD 임포트, 다채널 DSP 라우팅 등 Pro 핵심 기능은 포함 안 함.

**장점**
- 폰 화면 UX 문제 회피 (5밴드는 충분히 조작 가능)
- 구현량이 안 A보다 작음 — 기존 AI TUNE 패널의 APPLY 파이프 재사용
- "Pro는 20밴드 + FRD + 측정마이크"로 차별화 유지

**단점**
- "왜 5밴드냐"는 설명이 필요 (사용자에게 제한처럼 보임)
- Pro 탭 분리 시 홈 화면 네비게이션 구조 변경 필요

**T-shirt 사이즈: S~M** (5밴드 PEQ 모델 + 간단한 슬라이더 UI)

---

### 안 C — "아예 안 함, 역할 분담 유지"

모바일은 **"룸 튜닝 + 크로스오버 추천" 수준에서 완성**. 수동 PEQ는 Pro에만.
대신 모바일에서 "Pro 앱으로 넘어가세요" 딥링크/QR을 노출해 Pro 전환 유도.

**장점**
- 제품 포지셔닝 가장 깔끔 (혼선 없음)
- 코드베이스 분리 유지 → 유지보수 부담 최소
- Pro 라이선스 가치(₩99,000) 훼손 없음
- 폰 마이크 한계를 "Pro 측정 마이크가 필요한 이유"로 마케팅 포인트화 가능

**단점**
- 모바일 앱만 쓰는 사용자는 미세 조정 불가 → 이탈 가능성

**T-shirt 사이즈: XS** (딥링크 버튼 하나)

## 3. 추천안 및 이유

**→ 안 B (Pro Lite 탭, 5밴드 PEQ)**를 추천한다.

**이유:**

1. **폰 마이크 한계 때문에 안 A의 20밴드는 의미 없다.** 측정 정밀도가 ±3dB
   이상인 구간에서 1/3 옥타브 정밀 PEQ를 조작하면 오히려 음질이 나빠진다.
   5밴드 광대역 보정은 폰 마이크로도 신뢰할 수 있는 범위다.

2. **Pro 라이선스 차별화를 지킬 수 있다.** 5밴드는 "맛보기", 20밴드 + FRD +
   전문 마이크 연동은 Pro만 — 이 구도가 성립한다.

3. **지금 당장 구현 가능하다.** AI TUNE 패널이 이미 밴드별 APPLY 파이프를 가지고
   있으므로, 5밴드 슬라이더 UI만 붙이면 된다. dsp_engine 전체 이식 불필요.

**단, 지금 구현 안 하는 이유:**

현재 1단계(보드 자동탐지)와 모바일 FRD 임포트가 미완인 상태에서 PEQ UI를
추가하면 "측정도 제대로 안 되는데 수동 PEQ가 있다"는 어색한 UX가 된다.
**선행 조건: 1단계 ADAU1466 어댑터 완성 + 모바일 FRD 임포트 완성 후 착수.**
